## 4. 本轮发现并修复的问题

| 问题 | 修复与 UI 复验结果 |
|---|---|
| 经营驾驶舱任一跨部门接口 403 会导致整页归零 | 各明细独立按权限加载，公司 BI 独立展示；总经办刷新后显示真实金额 |
| 权限外明细被显示成“0、达标、平稳” | 改为“数据受限/无明细权限”，健康度显示 `--`，避免误导 |
| 员工工作台调用无权访问的人事统计并显示 0 人 | 移除越权统计依赖，改为个人请假与个人档案真实数据 |
| 未来到期应收未进入“未到期”账龄 | 修正逾期天数符号和区间；页面显示 `¥336,000.00` |
| 客户开票没有生成收入确认凭证 | 开票生成“借：应收账款，贷：主营业务收入”，历史数据迁移补齐 |
| 供应商发票未入账导致应付为负 | 发票审核生成“借：库存商品，贷：应付账款”，历史数据迁移补齐 |
| 项目退料缺少原因和库存恢复不完整 | 补充退料原因字段、服务逻辑和页面展示，库存回归通过 |
| 多角色菜单、接口权限和数据范围不一致 | 修正角色权限、后端校验与前端菜单；各角色重新登录验证 |
| 审批节点、撤回、转交/加签及待办展示不一致 | 统一审批运行节点与待办查询，真实审批流程复验通过 |
| 采购、项目、资质、审计等页面存在字段或跳转断点 | 修正接口字段、详情跳转与业务轨迹展示，逐页复验通过 |
